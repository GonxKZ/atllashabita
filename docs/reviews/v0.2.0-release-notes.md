# AtlasHabita · Release Notes v0.2.0

Fecha objetivo: abril 2026 · hito M8 (ingesta real + UI nacional + SPARQL + release).

Esta release cierra las cuatro fases del plan M8 y marca el salto del "demo con 10 municipios" a una plataforma **demostrable a escala nacional** con procedencia completa, panel técnico y capa de exportación RDF.

## Resumen ejecutivo

- **Dataset nacional mock**: >=100 municipios reales (capitales y cabeceras de comarca) con indicadores plausibles en orden de magnitud (INE, MITECO, Sanidad, AEMET).
- **Ranking ampliado** con paginación 20/página, dos filtros duros (precio máximo de alquiler y conectividad mínima) y badge de confianza.
- **Ficha territorial completa** con cabecera contextual, tabla de indicadores, chips PROV-O con licencia/periodo/link oficial y modal "Ver RDF" paginado.
- **Panel de capas** con 6 capas activables a la vez (score, renta, alquiler, banda ancha, servicios, clima) y leyenda por indicador.
- **Panel SPARQL** con catálogo de consultas, validador tipado de bindings y ejecutor que cae en modo fallback local si el backend no está disponible.
- **Cliente SPARQL / RDF**: `services/sparql.ts`, `services/rdf_export.ts` + hooks TanStack Query (`useSparql`, `useRdfExport`).
- **Rutas nuevas**: `/ranking`, `/territorio/:id`, `/sparql` integradas en `AppRouter`. El panel SPARQL se carga con `React.lazy` para mantener el bundle inicial acotado.

Ubicados en `apps/web/src/components/ui/`:

- `Pagination.tsx` con `buildPageRange` determinista y elipsis.
- `Slider.tsx` accesible sobre `<input type="range">`.
- `CodeBlock.tsx` con numeración de líneas y copiar al portapapeles.

Todos exponen el contrato estándar `ComponentPropsWithoutRef` y se registran en el barrel `components/ui/index.ts`.

## Decisiones técnicas relevantes

### Validador de bindings SPARQL sin `zod`

El task restringe el alcance a ciertos directorios. Incorporar `zod` como dependencia implicaría tocar `package.json`, fuera del scope. Por eso se ha implementado un validador minimalista en `features/sparql/schema.ts` que emula la API `safeParse` y soporta coerción de tipos (`string`, `integer`, `number`, `boolean`, `iri`). La migración a `zod` es un renombre tras añadir la dependencia en una iteración futura.

### Fallback local visible

Tanto el panel SPARQL como la ficha territorial incluyen un **fallback determinista** si el backend real de Fase C aún no está operativo:

- SPARQL: se ejecuta sobre el dataset mock nacional y se devuelve el mismo shape (`SparqlResult`).
- RDF: se construye un Turtle sintético con prefijos `ah:`, `dct:`, `prov:` y `xsd:` y se pagina en bloques de 20 líneas.

Esto permite grabar vídeos, hacer capturas y demos TFG aunque la API esté caída, sin ocultar la ausencia: ambos paneles muestran un `Tag tone="warning"` con el texto "Fallback local" o "Catálogo local".

### Selección sincronizada ranking ↔ mapa

`RankingPanel` usa el store de Zustand `state/selection.ts`. Al pulsar un item, se actualiza `selectedTerritoryId`; el mapa y cualquier otra vista que consuma ese store pueden centrar/destacar esa entrada sin dependencias cruzadas.
