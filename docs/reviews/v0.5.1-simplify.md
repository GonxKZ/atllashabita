## Detalle de la simplificacion APLICADA

### 1. Extraer cliente GitHub a `_github_api.py`

**Antes**: cada uno de los cuatro scripts `create_m{8,9,11,12}_issues.py` duplicaba
~30 lineas (importaciones de `urllib`, `json`, definicion de `TOKEN`/`API`/`call`/
`find_milestone`/`existing_titles`).

**Ahora**: `scripts/_bootstrap/_github_api.py` expone una clase `GithubApi` con
`call`, `find_milestone` y `existing_titles`. Los cuatro scripts se reducen a:

```python
from _github_api import GithubApi
api = GithubApi(repo="GonxKZ/atllashabita")
```

**Impacto**:
- 4 scripts mucho mas cortos (de ~50 a ~15 lineas de boilerplate cada uno).
- Comportamiento HTTP identico: misma cabecera Authorization, mismo tratamiento
  de `urllib.error.HTTPError` (devuelve `{"error": code, "body": ...}`).
- `create_issues.py` (script base original) NO se toca: su contrato strict
  (`raise RuntimeError`) es distinto y simplificarlo requeriria modo opcional
  en el helper.
- 0 regresion de comportamiento: los scripts siguen siendo idempotentes y
  conservan el mismo formato de salida (`+ #123 [titulo]`, `= existe`,
  `! error ...`).
